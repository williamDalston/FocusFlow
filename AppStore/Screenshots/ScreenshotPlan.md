## Screenshot Best Practices

### Design Principles
1. **Show, Don't Tell**: Let visuals demonstrate features
2. **Clear Hierarchy**: Most important feature first
3. **Consistent Style**: Use same design language as app
4. **Readable Text**: Ensure overlay text is legible
5. **Real Data**: Use realistic workout data (not placeholder)

### Text Overlays
- Keep text concise and impactful
- Use large, readable fonts
- Ensure good contrast with background
- Limit to 1-2 lines per screenshot
- Use action-oriented language

### Visual Elements
- Use the app's actual UI (not mockups)
- Show real workout data
- Highlight key features clearly
- Use consistent color scheme
- Ensure proper spacing and alignment

### Technical Requirements
- Use actual device screenshots (not simulators if possible)
- Ensure proper resolution for each device size
- No status bar issues (use clean screenshots)
- Proper aspect ratios
- No UI glitches or errors

## Screenshot Creation Workflow

### Step 1: Prepare App
- Ensure app is in release-ready state
- Create demo data/scenarios for screenshots
- Test all features to be showcased
- Verify UI looks polished

### Step 2: Capture Screenshots
- Use actual devices or high-quality simulators
- Capture each screenshot at full resolution
- Take multiple variations for each screen
- Ensure consistent lighting/style

### Step 3: Edit & Enhance
- Add text overlays if needed
- Ensure consistent styling
- Verify all text is readable
- Check for any UI issues

### Step 4: Optimize
- Verify correct resolutions
- Compress if needed (while maintaining quality)
- Organize by device size
- Name files clearly

### Step 5: Test
- Upload to App Store Connect
- Preview on different devices
- Verify all screenshots display correctly
- Get feedback before finalizing

## Minimum Screenshot Requirements

**Required for Submission:**
- At least 1 screenshot for each device size
- Must show actual app functionality
- Cannot be just marketing images
- Must match app's current version

**Recommended:**
- 5-10 screenshots per device size
- Show different features and flows
- Tell a story about the app
- Highlight key differentiators

## Device-Specific Considerations

### iPhone
- Portrait orientation only
- Show key features prominently
- Ensure text is readable on smaller screens
- Highlight mobile-specific features

### iPad
- Portrait or landscape orientation
- Show how app utilizes larger screen
